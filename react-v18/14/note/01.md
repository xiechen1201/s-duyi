# 高阶组件

HOC，逻辑复用的技巧。

并不是一个组件，而是增强组件功能的一个函数。

## 高阶组件事一个函数

参数作为组件，返回值为一个新组件的函数。

## 组件要做的事情

公共逻辑的复用。

早期使用 minxins，现在废弃，存在问题：

- 隐式的依赖关系

- 名称冲突

- 滚雪球的复杂性

高阶组件还可以进行嵌套操作，例如两端公共逻辑。

```tsx
const NewChildCom1 = withLog(ChildCom1);
const NewChildCom2 = withLog(ChildCom2);

const NewChildCom1WithTimer = withTimer(NewChildCom1);
const NewChildCom2WithTimer = withTimer(NewChildCom2);
```

## 高阶组件的现状

HOC 早期就是为了解决类组件公共逻辑抽离的问题。但是现在 hook 的流行，公共逻辑都变为 hook 来实现。
