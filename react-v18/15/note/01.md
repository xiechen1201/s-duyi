# Ref

## string 类型的 refs

this.refs.inputRef

最早的用法，现在已经废弃。

ref 的注意点：

- 避免使用 refs 来完成声明式可以完成的操作

因为 ref 相当于会到 jquery 时代，现代框架都是声明式。

某些时候么办法才可以使用。

- 这个 API（字符串形式）已废弃

## createRef

ref 也可以进行挂在组件上，然后就可以调用组件内部的方法（依然是反模式）。

默认函数组件不能使用 ref，因为没有实例。

## Ref 转发

一个可选的特性。

高阶组件对组件进行封装，无法调用目标组件中的方法，因此需要进行 ref 转发。

## useRef & useImperativeHandle

函数组件的调用。

React.createRef() 每次渲染都会创建一个新的 ref。

useRef() 则不会，和 fibar 架构有关系。

> 设置一个 counter 状态，自增的时候观察 ref 的变化。

useRef 还可以存在不需要变化的全局变量非常方便。
